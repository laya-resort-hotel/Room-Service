# LAYA Room Service QR v2.4 - Firebase Storage

เวอร์ชันนี้เพิ่มระบบอัปโหลดรูปเมนูเข้า **Firebase Storage** แล้วเก็บเฉพาะ URL ของรูปไว้ใน Firestore
ทำให้ฐานข้อมูลเบากว่าแบบ Base64 และเหมาะกับงานจริงมากขึ้น

## ไฟล์สำคัญ
- `index.html` หน้าลูกค้า
- `staff.html` หน้าพนักงานรับออเดอร์/แชท
- `admin.html` หน้าแอดมินเพิ่ม/แก้ไขเมนู 4 ภาษา + อัปโหลดรูป
- `qr.html` หน้าสร้าง QR ตามเลขห้อง
- `firebase-config.js` ตั้งค่า Firebase
- `firestore-rules.txt` Rules สำหรับ Firestore
- `storage-rules.txt` Rules สำหรับ Firebase Storage

## วิธีอัปเดตบน GitHub
อัปโหลดไฟล์ทั้งหมดใน ZIP นี้ทับของเดิมใน repo `Room-Service` แล้วรอ GitHub Pages deploy 1-3 นาที

ถ้าต้องอัปโหลดเฉพาะไฟล์สำคัญ ให้ทับไฟล์เหล่านี้:
- `admin.html`
- `admin.js`
- `firebase-service.js`
- `styles.css`
- `storage-rules.txt`

## ต้องเปิด Firebase Storage
1. เข้า Firebase Console
2. เลือกโปรเจกต์ `roomservice-3f58c`
3. ไปที่ Build > Storage
4. กด Get started / Create bucket
5. ไปที่แท็บ Rules
6. คัดลอกจาก `storage-rules.txt` ไปวาง แล้วกด Publish

## วิธีใช้
1. เข้า `admin.html`
2. เพิ่มเมนูหรือกดแก้ไขเมนูเดิม
3. เลือกรูปจากเครื่องในช่อง “อัปโหลดรูปภาพไป Firebase Storage”
4. กดบันทึก
5. ระบบจะย่อรูปก่อน แล้วอัปโหลดเข้า Storage จากนั้นบันทึก URL ลง Firestore

## ย้ายรูป Base64 เดิม
ถ้าเคยบันทึกรูปแบบ Base64 ไว้แล้ว ให้เข้า `admin.html` แล้วกดปุ่ม:

**ย้ายรูป Base64 ไป Storage**

ระบบจะไล่ย้ายรูปเดิมทีละรายการ แล้วอัปเดต URL ให้เอง

## หมายเหตุความปลอดภัย
Rules ที่ให้มาเป็นแบบเปิดสำหรับทดสอบเท่านั้น งานจริงควรเพิ่ม Firebase Authentication และจำกัดสิทธิ์ให้เฉพาะ Admin/Staff เท่านั้น

## v2.6 Auto Translate

เพิ่มเครื่องมือแปลภาษาอัตโนมัติในหน้า `admin.html`

- พิมพ์ชื่อเมนูและรายละเอียดภาษาไทย ระบบจะช่วยแปลเป็น English / 中文 / Русский อัตโนมัติ
- ปุ่ม `แปลช่องที่ว่างในฟอร์มนี้` ใช้เติมเฉพาะช่องที่ยังว่าง
- ปุ่ม `แปลใหม่ทับภาษาอื่น` ใช้แปลใหม่ทับช่องภาษาอื่นในฟอร์มเดียว

หมายเหตุ: ระบบแปลอัตโนมัตินี้เป็นตัวช่วยสำหรับต้นแบบ ควรให้พนักงานตรวจคำแปลก่อนใช้งานจริง โดยเฉพาะชื่อเมนู วัตถุดิบ และข้อมูลแพ้อาหาร

หมายเหตุ: ระบบนี้ใช้ `chats/{room}/messages` และใช้ collection group เพื่อทำ inbox รวม ในงานจริงที่ข้อความเยอะมาก แนะนำเพิ่ม collection `chatRooms` สำหรับเก็บ last message โดยเฉพาะเพื่อให้โหลดเร็วขึ้น


## v3.1 Chat Inbox Fix

แก้ปัญหา Staff Chat Inbox ค้างที่ "กำลังโหลด..." หรือไม่เห็นข้อความลูกค้า โดยเปลี่ยนระบบกล่องข้อความเข้าให้ใช้ collection `chatRooms` แทนการอ่านแบบ collectionGroup จาก `chats/*/messages`.

หลังอัปเดตต้อง Publish Firestore Rules ใหม่ด้วย เพราะเพิ่ม collection:

```js
match /chatRooms/{roomId} {
  allow read, write: if true;
}
```

หลังอัปโหลดไฟล์และ Publish Rules แล้ว ให้ลูกค้าส่งข้อความใหม่อีกครั้ง เช่น "ทดสอบ" ห้องนั้นจะขึ้นในกล่องข้อความเข้าทันที


## v3.2 Staff Chat Alert

เวอร์ชันนี้เพิ่มการแจ้งเตือนแชทบนหน้า `staff.html`:

- เมื่อลูกค้าทักมา จะมีป๊อปอัปเด้งบน Staff Board ทันที
- มีเสียงแจ้งเตือนซ้ำเบา ๆ เพื่อกันพนักงานไม่ได้มองจอ
- แถบชื่อแท็บจะกระพริบเป็นเลขห้องที่ทักมา
- ถ้า Staff ใช้ Chrome/Edge และกดปุ่ม “เปิดเสียง/แจ้งเตือน” จะขออนุญาต Desktop Notification เพิ่มด้วย
- กดปุ่ม “เปิดแชท” บนป๊อปอัปเพื่อเข้าไปตอบห้องนั้นทันที

หมายเหตุ: การแจ้งเตือนนี้ทำงานเมื่อหน้า `staff.html` เปิดค้างอยู่ในเครื่องพนักงาน หากปิด browser หรือปิดคอม จะไม่สามารถเด้งแจ้งเตือนได้จนกว่าจะทำระบบ Push Notification/พนักงาน Login เพิ่มในเวอร์ชันถัดไป

## v3.5 เปลี่ยนรูป Cover หน้าแรก

เพิ่มหน้า Admin สำหรับแก้ไขรูป Cover ด้านบนของหน้าลูกค้าได้เอง

วิธีใช้:
1. เข้า `admin.html`
2. ไปที่กล่อง **ตั้งค่ารูป Cover หน้าแรก**
3. ใส่ URL รูปภาพ หรือเลือกรูปจากเครื่อง
4. กด **บันทึกรูป Cover**
5. หน้า QR ลูกค้าจะเปลี่ยนรูป Cover ตามแบบ Realtime

ไฟล์ที่เกี่ยวข้อง: `admin.html`, `admin.js`, `customer.js`, `firebase-service.js`, `styles.css`, `firestore-rules.txt`, `storage-rules.txt`

ต้องเปิด Firebase Storage และเพิ่ม Rules ให้ `site-images/{allPaths=**}` อ่าน/เขียนได้สำหรับช่วงทดสอบ
